# 디토챗(DittoChat) 기획 고도화용 마스터 프롬프트

## [역할 설정]
너는 혁신적인 소셜 서비스를 설계하는 시니어 앱 서비스 기획자이자 풀스택 개발자야. 내가 구상 중인 '디토챗(DittoChat)'이라는 서비스의 아이디어를 구체화하고, 기술적인 실현 방안과 상세 기능을 설계해줘.

## [서비스 개요]
* **서비스명:** 디토챗 (DittoChat)
* **핵심 컨셉:** 사용자의 MBTI와 자기소개를 바탕으로 생성된 'AI 아바타 봇(디지털 트윈)'이 다른 사용자의 아바타 봇과 대화하는 것을 사용자가 '관찰'하고 학습시키는 AI 페르소나 소셜 플랫폼.
* **슬로건:** "나보다 더 나다운 분신, 디토와의 대화 관찰기"

## [주요 기능 및 로직]
* **페르소나 생성:** 사용자가 MBTI와 간단한 자기소개를 입력하면 AI가 해당 성격의 말투, 가치관, 반응 로직을 가진 아바타를 생성함.
* **관찰형 채팅:** 사용자가 직접 채팅하는 것이 아니라, 내 아바타를 '육아 공유방', '또래 모임', '관심사 방' 등에 입장시켜 다른 봇들과 대화하는 과정을 실시간으로 지켜봄.
* **상성 매칭:** MBTI 궁합을 기반으로 '최고의 짝꿍' 혹은 '극상성'인 아바타와 대화 시나리오를 설정하여 관계를 시뮬레이션함.
* **피드백 및 학습:** 아바타의 대화 중 "나다운" 부분과 "아닌" 부분을 사용자가 체크하거나 직접 수정(빙의하기)하여, 아바타가 점점 더 사용자의 실제 성격과 동기화되도록 학습시킴.
* **가상 메이트 (Instant Match):** 초기 유저가 대기하는 지루함을 없애기 위해, 사용자가 원하는 상대(MBTI, 성격)를 선택하는 **즉시(0초)** 해당 페르소나의 AI 봇이 생성되어 대화를 시작함.

## [요청 사항]
위 내용을 바탕으로 다음 항목들을 상세히 제안해줘:
1. **아이디어 스트럭쳐:** 이 앱의 핵심 사용자 여정(User Journey)과 데이터 흐름을 설계해줘.
2. **기능 상세:** MVP(최소 기능 제품) 단계에서 반드시 필요한 'Must-have' 기능 5가지를 리스트업해줘.
3. **MBTI 로직:** MBTI의 각 지표(E/I, S/N, T/F, J/P)가 아바타의 대화 스타일에 구체적으로 어떤 영향을 주는지 예시를 들어 설명해줘.
4. **수익 모델:** 이 서비스가 지속 가능하기 위한 비즈니스 모델(BM) 아이디어를 제안해줘.
